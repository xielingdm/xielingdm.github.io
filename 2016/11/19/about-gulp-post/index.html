gulp的配置及使用教程

Veröffentlicht am 2016-11-19

###先说一个npm的概念
* npm – node提供的供下载的地方相当于应用商店

##一、安装gulp之前先安装配置nodejs >
1.安装node.js 直接next下一步，在cmd输入 node -v 如果能出现类似 v6.0.0 版本号则node安装成功

##二、安装gulp >

###第一种方式：

###第一步：安装淘宝镜像 ：

npm install -g cnpm –registry=https://registry.npm.taobao.org

因为npm安装插件是从国外服务器下载，受网络影响大，可能出现异常，如果npm的服务器在中国就好了
淘宝镜像可装可不装(国内服务器) cnpm -v 查看
注：cnpm跟npm用法完全一致，只是在执行命令时将npm改为cnpm（以下操作将以cnpm代替npm）。

###第二步：利用Cnpm 工具对gulp进行全局安装 ： Cnpm install gulp -g

###第三步 ： 输入gulp -v 查看版本号,出现版本号即为正确安装

###第四步：
进入需要的文件夹
例如： 在D盘(随便哪个盘)建立一个文件夹gulp
在cmd中输入 D： cd gulp进入这个文件夹

###第五步 ： 安装局部环境gulp ： cnpm install gulp

###第六步 ：创建package.json文件，输入 cnpm init 敲回车 -
在路径 下会形成这样的一个.json文件（部分内容为后添加的）
{

}
查看package.json帮助文档，命令提示符执行cnpm help package.json

###第七步：本地安装gulp插件
http://gulpjs.com/plugins/ 插件官网。

将插件安装到本地之后， 要require之后才可以正常使用插件。
下面以gulp-less为例：(更多插件将在下面展示)

—将会安装在 node_modules 的gulp-less目录下，该目录下有一个gulp-less的使用帮助文档README.md;

2.创建文件 在目录中创建src文件夹，在该文件夹中创建less文件夹和css文件夹
在less文件中创建一个index.less文件，写一些代码

3.在项目的根目录下创建一个 gulpfile.js 文件
在该文件写入

4.然后打开cmd窗口，或者你安装了git环境的话，直接在该文件夹下右键点解Git Base Here，然后输入gulp命令，那么你就在你的src/css文件夹下就可以看到一个index.css文件，这个文件就是我们编译less文件之后生成的一个css文件。

###第八步：gulpfile.js 配置
1.gulp.task( ) 方法，这个方法接收两个参数。 gulp.task （‘指令’，‘function函数‘）；

首先引入gulp,利用require引入。
var gulp = require(‘gulp’);

接下来为gulp配置一个hello指令
gulp.task(‘hello’,function(){
console.log(‘hello，world’)；
})

当我在命令行中输入gulp hello 的时候，那么我们的gulp会替我们执行 这个函数， 我们在这个函数上console.log了一个 hello 。 执行结果如下:

第二个很有用的命令是default,这不是一个自定义的指令了。而是代表某个指令的执行， defualt 代表默认情况下执行[‘hello’]代表的是要执行哪个函数

gulp.(‘default’,[‘hello’]);

在命令行中执行 gulp,并不使用命令名的时候， 会执行默认的函数， 也就是hello函数。

##2.用gulp进行文件拷贝
先学习三个方法
gulp.src(‘’)这个方法代表找到源文件路径。

.pipe() 这个方法用来执行gulp 中的各种方法。

gulp.dest(‘dist’)//代表将文件保存在哪里。

注意，gulp 是允许连缀的， 在这里我们用连缀的写法去编辑我们的命令。

var gulp = require(‘gulp’);

gulp.task(‘index’,function(){

//找到源文件      pipe()//使用功能 可以传入一个参数作为目标    gulp.dest()会保存到对应文件夹
return gulp.src('index.html').pipe(gulp.dest('dist'))

});

在这个语句执行结束之后那， 我们在命令行中得到的结果是:

会告诉你执行结束的时间，会告诉你执行的结果。

我们在项目文件夹中可以看到 多出了一个文件夹，文件夹下有一个拷贝过来的index.html。

命令行中， 输入 index 这个时候就可以copy 到dist文件夹下了。

##3.用gulp进行图片拷贝
项目应用， 我们可以通过刚刚学习的拷贝命令去拷贝其他项目中的文件， 包括项目中的图片

gulp.task(‘images’,function(){

return gulp.src('images/*.jpg').pipe(gulp.dest('dist/images'))//*代表所有

})

拷贝多种文件及文件夹：
1.拷贝多种后缀名 将 gulp.src(‘images/*.{jpg,png}’)写入文件中

2.如果要拷贝文件夹下所有内容，那么可以 将gulp.src(‘images//‘) 写入文件中 可以成功进行拷贝。 如果要拷贝所有的话， 可以使用 images/*/ 去表示当前images文件夹下所有的内容全部进行拷贝。

##4.多文件合并
将两个文件夹拷贝到同一个目标文件夹下。 这个时候那， 我们将文件源叫做glub ， 那么这个时候我们拷贝两个及两个以上源的时候那， 叫做多个源文件的拷贝。

我们需要更改的也就是 gulp.src([‘json/.json’,’xml/.xml’]);

这样我们就可以对文件进行一个更改了

gulp.task(‘data’,function(){
return gulp.src([‘xml/.list’,’json/.json’]).pipe(gulp.dest(‘dist/data’))
})

##5.排除 ！

return gulp.src([‘xml/.xml’,’json/.json’,’!json/secret.json’]).pipe(gulp.dest(‘dist/data’))
返回除了json文件夹下的secret.json文件的所有文件

##6.任务合并执行；（如何将以上命令一次性执行）

gulp.task(‘build’,[‘index-copy’,’image-copy’,’data’],function(){

console.log('编译成功')

})

##7.侦测文件变化， 自动执行任务；

gulp.watch()

gulp.task(‘watch’,function(){
gulp.watch(‘index.html’,[‘index-copy’]);
gulp.watch(‘images/*/‘,[‘images-copy’]);
gulp.watch([‘json/.json’,’xml/.xml’,’!json/secret.json’],[‘data’]);
//更改的文件 //执行的方法
})

侦测自动执行，如果文件发生改变那么将执行对应任务。

注：(假如要构建第二个项目，
1.新建一个文件夹，同样配置局部gulp环境，执行以上的局部gulp环境步骤
2.或者，在gulpfile.js文件下新建命令
例如，有了一个style.less文件

gulp.task(‘less’,function(){
return gulp.src(“src/less/style.less”)

现在要在创建一个style1.less文件，
gulp.task(‘less1’,function(){

##gulp插件

###1.gulpsass 即时sass编译插件

npm install gulp-sass 就可以使用了。

现在环境中引入gulp-sass > var sass = require(‘gulp-sass’);

gulp.task(‘sass’,function(){
return gulp.src(‘stylesheet/*/.scss’).pipe(sass()).pipe(gulp.dest(‘dist/css’))
})
这样我们调用 sass命令就可以对相应的代码进行编译了。

###2.gulpless 即时less编译插件

npm install gulp-less 就可以使用了。

现在环境中引入gulp-less > var less = require(‘gulp-less’);

gulp.task(‘less’,function(){
return gulp.src(‘stylesheet/*/less’).pipe(less()).pipe(gulp.dest(‘dist/css’))
})

这样我们调用 sass命令就可以对相应的代码进行编译了。

###3.gulp 创建本地服务

gulp connect

npm install gulp-connect 就可以使用了。

配置端口号， 服务器名称；

gulp.task(‘server’,function(){
connect.server({
root:’dist’, //以谁为服务器根目录
port:8888 // 端口号
});
})

默认端口：Server started http://localhost:8080

修改文件后自动刷新。

gulp.task(‘server’,function(){
connect.server({
root:’dist’
});
})

这时候我们需要开启服务器的同时，再去执行 watch那么我们这时候需要创建一个默认的任务；

如果需要实时更新数据，那么需要添加一个参数livereload:true
gulp.task(‘server’,function(){

connect.server({
    root:'dist',
    livereload:true
});

})

gulp.task(‘index-copy’,function(){

return gulp.src('index.html').pipe(gulp.dest('dist')).pipe(connect.reload());

})
gulp.task(‘default’,[‘server’,’watch’])

###4.gulp合并插件 gulp-concat

用于合并js文件；

gulp.task(‘scripts’,function(){
return gulp.src([‘scripts/hz.js’,’scripts/index.js’]).pipe(concat(‘vendor.js’)).pipe(gulp.dest(‘dist/js’))
})

###5.gulp代码压缩插件 gulp-uglify ：

gulp.task(‘scripts’,function(){
return gulp.src([‘scripts/hz.js’,’scripts/index.js’]).pipe(concat(‘vendor.js’)).pipe(uglify()).pipe(gulp.dest(‘dist/js’))
})

我们是否可以保存两份js文件，一份已经压缩，另一份没有压缩的那？

###6. 重命名 ； gulp rename插件；

gulp.task(‘scripts’,function(){
return gulp.src([‘scripts/hz.js’,’scripts/index.js’])
.pipe(concat(‘vendor.js’))
.pipe(gulp.dest(‘dist/js’))
.pipe(uglify())
.pipe(rename(‘vendor.min.js’))
.pipe(gulp.dest(‘dist/js’))
})

###7.css压缩 gulp-minify-css

gulp.task(‘sass’,function(){
return gulp.src(‘stylesheet/*/.scss’).pipe(sass()).pipe(minifyCSS()).pipe(gulp.dest(‘dist/css’))
})

###8. 图片压缩； gulp-imagemin插件；

gulp.task(‘image-copy’,function(){

return gulp.src('images/**/*').pipe(imagemin()).pipe(gulp.dest('dist/images'))

})
