Git 和 SVN

Veröffentlicht am 2016-10-02

git 和 svn 都是协同办公工具 也可以说是 版本管理工具

snv ： 分支型版本管理工具 也叫集中型版本管理工具

git ： 分布式版本管理工具

—-他俩都是免费的版本管理工具

svn 使用

1.先下载安装 svn ， 在重启电脑

2.引入服务器地址 并添加用户名密码验证 指令： svn checkout

3.代码更新 svn update

4.代码提交 svn commit

修改代码和服务器发生冲突的解决：

1. 先把所有index页面删掉

1. 在update更新 – 这时候down下来的就是最新的版本了

1. 然后在修改代码 – commit提交

git

git 是现在比较流行的分布式版本管理工具

优点：

1、分布式

2、和github社区交互

###分布式和集中式：

集中式：都分布在服务器上 需要哪个下哪个 没网不行

分布式：最大的好处是可以在无网的情况下工作 并且分支功能十分强大

git 是基于linux或者unix系统下进行开发的

在linux和unix系统下使用的软件

如果在windows上使用cygwin（在windows下可以输入linux指令）进行模拟

msysgit – 专门给windows开发的git 在windows 我们使用msysgit 进行开发

git使用步骤

1、第一步 安装git 如果是windows系统 安装msysgit

2、第二步 右键进入git bash

3、初始化配置用户名 和 邮箱 在命令窗口中输入

git config –global user.name ‘自己的名字’

git config –global user.email ‘自己的邮箱’

4、创建仓库 选的安装目录 我选择在D盘新建目录 并进入cd /d
创建目录 – mkdir 还可以用pwd进行查看

5、跳转到指定目录 进行仓库初始化 git init

6、假设先建立了一个readme.txt

如果想要提交 要先把他放到暂存区中 使用命令 git add readme.txt – 相当于把readme.txt放入暂存区中

一次可以add多个文件

7、提交命令 git commit 进行文件的提交

git commit readme.txt -m “you must read it before you work” 》》 -m 添加注释的

8、如果我们需要修改 git status 查看状态

9、使用 git diff readme.txt – 查看文件跟上次有什么不同 修改的细节

10、如果觉得没什么问题

git add readme.txt – 第二次提交到暂存区

git commit readme.txt -m “this is my second debug” – 第二次提交到工作区

12、若我们需要找回历史版本 查看版本 使用 git log

13、回归版本 我们使用 git reset –hard HEAD^

^代表回归的第几个版本 ^上个版本 ^^上上个版本 ^^^上两个版本

14、选的某个版本 我们使用 git reset –hard “该版本的commit ID”
如果回归了版本又想找回现在的版本 需要commit ID 如果关闭了窗口 commit的ID没有了
现在的版本就找不回了 所有慎用回归版本

15、真实项目中远程连接 github

github 的https 是windows使用的 ssh是linux和unix使用的

git remote add origin https://github.com/'github账户名称'/'名称下的一个仓库‘

16、真正提交到远程仓库当中了

git push -u origin master

第二次提交不要加 -u

17、下载到本地

git clone https://github.com/'github账户名称'/'名称下的一个仓库‘
