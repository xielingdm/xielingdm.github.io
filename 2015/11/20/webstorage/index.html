啥是web Storage?

webStorage是一种新的web存储数据技术， 非常类似于cookie。 因为cookie对储存的容量以及多种问题， 所以HMLT4推出了类似cookie究极进化版的localStorage。

localStorage官方给出的极限储存数据是5M。

为何cookie被吐槽的如此厉害那：

大小：cookies的大小限制在4KB

带宽：cookies是随HTTP事务一起被发送的，因此会浪费一部分带宽

复杂性：正确的操纵cookies是很困难的

localStorage分为两种：

sessionStorage

localStorage

sessionStorage 将数据保存在浏览器的session对象之中， 用作临时储存，安全性较高， 一般用于银行等地的数据存储。

localStorage 将数据存在本地硬件设备中，可以永久保存，除非主动删除，否则永远不会删除。

localStorage和sessionStorage的可用API大多相同。

  • 保存数据：localStorage.setItem( key, value ); sessionStorage.setItem( key, value );
  • 读取数据：localStorage.getItem( key ); sessionStorage.getItem( key );
  • 删除单个数据：localStorage.removeItem( key ); sessionStorage.removeItem( key );
  • 删除所有数据：localStorage.clear( ); sessionStorage.clear( );
  • 得到某个索引的key：localStorage.key( index ); sessionStorage.key( index );

对于JSON数据的存储以及操作；

webStorage是只能存储字符串的。 但是对于复杂数据的操作我们必须要是用的是JSON，那么怎么在webStorage中保存和使用JSON数据那？

JSON.stringify(data) 将一个对象转换成字符串形式。 便于储存。
JSON.parse(string) 将数据解析成对象，返回解析后的对象。
